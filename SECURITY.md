# Security Policy

## Supported Versions

The following versions of EdgeMultiProfile are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅ Yes     |
| < 1.0   | ❌ No      |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

Submit a vulnerability report through one of the following channels:

* Open a private security advisory (if enabled)
* Contact the maintainers directly via email
* Create a confidential security report through the project's designated reporting channel

### Information to Include

Please provide:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Affected version(s)
* Proof-of-concept, if available

### Response Timeline

We aim to:

* Acknowledge reports within 72 hours
* Provide an initial assessment within 7 days
* Release a fix as soon as reasonably possible based on severity

### Responsible Disclosure

Please do not publicly disclose security vulnerabilities until a fix has been released and users have had reasonable time to update.

## Security Best Practices

Users are encouraged to:

* Always use the latest release.
* Verify release integrity using published SHA-256 checksums.
* Download binaries only from official release pages.
* Review release notes before upgrading.

## Security Updates

Security fixes will be documented in future release notes and changelogs. Critical vulnerabilities may result in out-of-band releases when necessary.
